# 📦 MindSpore 模型导出和转换指南

## ✅ 已修复的问题

### 原问题
```
ValueError: Only support export file in 'AIR' format with Ascend backend.
```

**原因**：AIR 格式只支持华为昇腾（Ascend）硬件，CPU/GPU 不支持。

### 解决方案
改用 **MINDIR** 格式，这是 MindSpore 的通用中间表示格式，支持所有后端。

## 🔧 转换为 MS 格式（Android 部署）

### 方法 1：使用 MindSpore Lite Converter（推荐）

#### 安装 Converter
```bash
# 下载 MindSpore Lite Converter
# 从 https://www.mindspore.cn/lite 下载对应平台的 converter

# 或者使用 pip
pip install mindspore-lite
```

#### 转换命令
```bash
# Windows
converter.exe --fmk=MINDIR --modelFile=style_classifier_artbench.mindir --outputFile=style_classifier

# Linux/Mac
./converter --fmk=MINDIR --modelFile=style_classifier_artbench.mindir --outputFile=style_classifier
```

**参数说明**：
- `--fmk=MINDIR`: 输入格式为 MINDIR
- `--modelFile`: 输入的 MINDIR 文件路径
- `--outputFile`: 输出的 MS 文件名（不含扩展名）

**输出**：
```
style_classifier.ms  # 转换后的模型文件
```

---

### 方法 2：使用 Python API 转换

创建转换脚本 `convert_to_ms.py`：

```python
import mindspore_lite as mslite

# 加载 MINDIR 模型
model = mslite.Model()
model.build_from_file("style_classifier_artbench.mindir", mslite.ModelType.MINDIR)

# 导出为 MS 格式
model.export("style_classifier.ms", mslite.ModelType.MINDIR)
print("✓ 转换完成: style_classifier.ms")
```

运行：
```bash
python convert_to_ms.py
```

### 方法 3：直接使用 MINDIR 格式

## 📱 部署到 Android

### 步骤 1：转换模型
```bash
converter --fmk=MINDIR \
          --modelFile=style_classifier_artbench.mindir \
          --outputFile=style_classifier
```

### 步骤 2：复制到 Android 项目
```bash
# 将 .ms 文件复制到 assets 目录
copy style_classifier.ms ..\custommodel\src\main\assets\
```

### 步骤 3：更新 Android 代码
在 `StyleClassifierExecutor.java` 中：

```java
// 加载模型
String modelPath = "file:///android_asset/style_classifier.ms";
boolean success = loadModel(modelPath);

if (success) {
    Log.i(TAG, "模型加载成功");
} else {
    Log.e(TAG, "模型加载失败");
}
```

### 步骤 4：编译并运行
```bash
cd ..
./gradlew assembleDebug
```

## 📊 格式对比

| 格式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **MINDIR** | 通用、跨平台 | 文件较大 | 开发、测试 |
| **MS** |  optimized for Lite | 需要转换 | Android/iOS 部署 |
| **AIR** | 昇腾专用 | 仅 Ascend | 昇腾硬件 |
| **ONNX** | 跨框架 | 可能有兼容性问题 | 其他框架互操作 |

## ⚠️ 常见问题

### Q1: 找不到 converter 工具
**A**: 从 MindSpore 官网下载 Lite 版本，包含 converter 工具。

### Q2: 转换失败
**A**: 检查：
1. MINDIR 文件是否完整
2. MindSpore Lite 版本是否与训练版本兼容
3. 输入形状是否正确

### Q3: Android 加载失败
**A**: 确保：
1. .ms 文件已正确复制到 assets 目录
2. 文件路径正确
3. MindSpore Lite 库已正确集成

### Q4: 模型太大
**A**: 使用量化：
```bash
converter --fmk=MINDIR \
          --modelFile=style_classifier.mindir \
          --outputFile=style_classifier_quant \
          --quantType=PostTrainingQuantization
```
