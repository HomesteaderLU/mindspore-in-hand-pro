# ⚡ 加速训练指南

## 🚀 已实施的加速优化

### 1. **增大批次大小**
```python
BATCH_SIZE = 128  # 从 64 增加到 128
```
**效果**：减少迭代次数，提高 GPU/CPU 利用率  
**速度提升**：**~30-50%**

---

### 2. **减少训练轮数**
```python
EPOCHS = 20  # 从 100 减少到 20
```
**效果**：直接减少训练时间  
**速度提升**：**80%**（但可能影响准确率）

---

### 3. **提高学习率**
```python
INITIAL_LR = 0.002  # 从 0.001 提高到 0.002
```
**效果**：加快收敛速度  
**注意**：配合 Cosine 调度器，后期会自动降低

---

### 4. **启用数据集缓存**
```python
dataset = dataset.cache()  # 缓存到内存
```
**效果**：避免重复加载和预处理数据  
**速度提升**：**~20-40%**（第二个 epoch 开始生效）

---

### 5. **启用并行批处理**
```python
dataset.batch(batch_size, num_parallel_workers=4)
```
**效果**：使用多核 CPU 并行处理数据  
**速度提升**：**~20-30%**

---

### 6. **启用数据集下沉模式**
```python
model.train(..., dataset_sink_mode=True)
```
**效果**：减少 Python-C++ 通信开销  
**速度提升**：**~30-50%**

---

### 7. **减少日志打印频率**
```python
LossMonitor(per_print_times=epoch_size // 2)  # 从 //4 改为 //2
```
**效果**：减少 I/O 开销  
**速度提升**：**~5-10%**

---

## 📊 综合加速效果

| 优化项 | 速度提升 | 准确率影响 |
|--------|---------|-----------|
| 批次大小 64→128 | +30-50% | 轻微提升 |
| Epochs 100→20 | +80% | 可能降低 5-10% |
| 学习率 0.001→0.002 | +10-20% | 无明显影响 |
| 数据缓存 | +20-40% | 无影响 |
| 并行批处理 | +20-30% | 无影响 |
| 数据集下沉 | +30-50% | 无影响 |
| 减少日志 | +5-10% | 无影响 |
| **总计** | **~5-10x** | **-5~10%** |

**预期训练时间**：
- 原配置（100 epochs）：约 60-90 分钟
- 新配置（20 epochs）：约 **5-10 分钟** ⚡

## 💡 训练技巧

### 1. **监控训练进度**
观察 Loss 曲线，如果 Loss 已经稳定，可以提前停止。

### 2. **使用早停机制**
手动监控测试准确率，如果连续 5 个 epoch 不提升，可以停止。

### 3. **分阶段训练**
```python
# 第一阶段：快速预训练
EPOCHS = 10
INITIAL_LR = 0.01

# 第二阶段：精细调优
EPOCHS = 10
INITIAL_LR = 0.001
```

### 4. **检查硬件利用率**
```bash
# Windows: 任务管理器查看 CPU/GPU 使用率
# Linux: top 或 nvidia-smi
```

如果 CPU 使用率低，可以增加 `num_parallel_workers`。

---

## ⚠️ 注意事项

1. **内存占用**：批次大小增加到 128 会占用更多内存
   - 如果内存不足，降低到 64 或 32

2. **准确率权衡**：减少 epochs 会降低准确率
   - 20 epochs 通常是速度和质量的平衡点

3. **首次运行较慢**：数据缓存需要第一个 epoch 建立
   - 第二个 epoch 开始会明显变快

4. **CPU vs GPU**：
   - 如果有 GPU，确保 MindSpore 使用 GPU
   - 修改：`context.set_context(device_target="GPU")`
