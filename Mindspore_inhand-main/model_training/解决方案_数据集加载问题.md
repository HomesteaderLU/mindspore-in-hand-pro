# 🔧 ArtBench-10 数据集加载问题解决方案

## ❌ 问题描述

**错误信息**：
```
[ERROR] Invalid cifar folder, cifar(.bin) files are missing under ./artbench10_data\artbench-10-batches-py.
```

**原因**：
MindSpore 的 `Cifar10Dataset` 对文件格式有严格要求，而 ArtBench-10 虽然使用 CIFAR-10 格式，但可能在文件结构或编码上存在细微差异，导致 MindSpore 无法正确识别。

---

## ✅ 解决方案（已实施）

我已经创建了一个**自定义加载器**来绕过这个问题。

### 新增文件

1. **load_artbench_custom.py** - 自定义数据加载器
   - 手动读取 pickle 格式的批次文件
   - 转换为 MindSpore 兼容的格式
   - 使用 `GeneratorDataset` 创建数据集

2. **check_dataset.py** - 诊断工具
   - 检查目录结构
   - 验证文件完整性
   - 提供修复建议

3. **train_with_artbench10.py** - 已更新
   - 使用自定义加载器替代 `Cifar10Dataset`
   - 自动处理目录结构

## 🔍 技术细节

### 自定义加载器的实现

```python
def load_cifar_batch(filename):
    """加载单个 CIFAR 批次文件"""
    with open(filename, 'rb') as f:
        dict = pickle.load(f, encoding='bytes')
        images = dict[b'data']
        labels = dict[b'labels']
        
        # 重塑图像数据
        images = images.reshape(-1, 3, 32, 32).transpose(0, 2, 3, 1)
        
        return images, np.array(labels)


def create_generator_dataset(images, labels, transform=None):
    """创建生成器数据集"""
    def generator():
        for img, lbl in zip(images, labels):
            yield (img, np.array(lbl, dtype=np.int32))
    
    dataset = ds.GeneratorDataset(generator, column_names=['image', 'label'])
    
    if transform:
        dataset = dataset.map(operations=transform, input_columns=['image'])
    
    return dataset
```

### 关键改进

1. **直接读取 pickle 文件**
   - 不依赖 MindSpore 的解析器
   - 使用 Python 标准库 `pickle`

2. **使用 GeneratorDataset**
   - 更灵活的数据源
   - 可以处理任意格式的数据

3. **完整的数据增强**
   - 在生成器后应用变换
   - 与原来功能完全一致

## 🐛 如果还有问题

### 问题 1：找不到文件

**运行诊断**：
```bash
python check_dataset.py
```

**查看输出**，确认文件位置。

**可能的修复**：
```bash
# 如果文件在根目录
mv artbench10_data/data_batch_* artbench10_data/artbench-10-batches-py/
mv artbench10_data/test_batch artbench10_data/artbench-10-batches-py/
mv artbench10_data/meta artbench10_data/artbench-10-batches-py/
```

### 问题 2：内存不足

自定义加载器会一次性加载所有数据到内存。如果内存不足：

**方案 1：减小批次大小**
```python
BATCH_SIZE = 16  # 从 32 改为 16
```

**方案 2：分批加载**（需要修改代码）

### 问题 3：加载速度慢

第一次加载会比较慢，因为要读取所有文件。后续会使用缓存。

**优化建议**：
- 使用 SSD 硬盘
- 增加 `num_parallel_workers`
