# 🔧 ArtBench-10 数据集问题排查指南

## ❌ 常见错误及解决方案

### 错误 1: "Invalid cifar folder, cifar(.bin) files are missing"

**错误信息**：
```
[ERROR] Invalid cifar folder, cifar(.bin) files are missing under ./artbench10_data.
```

**原因**：目录结构不正确，MindSpore 找不到 `data_batch_1` 等文件。

**解决方案**：

#### 方法 1：检查并修正目录结构

ArtBench-10 解压后应该有正确的目录结构：

```
artbench10_data/
└── artbench-10-batches-py/
    ├── data_batch_1
    ├── data_batch_2
    ├── data_batch_3
    ├── data_batch_4
    ├── data_batch_5
    ├── test_batch
    └── meta
```

**检查当前结构**：
```bash
ls -la artbench10_data/
```

**如果文件直接在根目录**：
```bash
# 创建子目录并移动文件
mkdir -p artbench10_data/artbench-10-batches-py
mv artbench10_data/data_batch_* artbench10_data/artbench-10-batches-py/
mv artbench10_data/test_batch artbench10_data/artbench-10-batches-py/
mv artbench10_data/meta artbench10_data/artbench-10-batches-py/
```

#### 方法 2：重新下载和解压

```bash
# 删除现有数据
rm -rf artbench10_data

# 重新下载
python download_artbench10.py
```

#### 方法 3：手动验证文件存在

```bash
# 检查必需的文件
ls artbench10_data/artbench-10-batches-py/data_batch_1
ls artbench10_data/artbench-10-batches-py/data_batch_2
ls artbench10_data/artbench-10-batches-py/data_batch_3
ls artbench10_data/artbench-10-batches-py/data_batch_4
ls artbench10_data/artbench-10-batches-py/data_batch_5
ls artbench10_data/artbench-10-batches-py/test_batch
ls artbench10_data/artbench-10-batches-py/meta
```

所有文件都应该存在。

### 错误 2: "FileNotFoundError: 找不到 ArtBench-10 数据文件"

**原因**：数据集未下载或路径错误。

**解决方案**：

1. **确认数据集已下载**：
```bash
ls -lh artbench10_data/
```

应该看到约 2GB 的文件。

2. **运行验证脚本**：
```bash
python download_artbench10.py --verify
```

3. **如果未下载，重新下载**：
```bash
python download_artbench10.py
```

---

### 错误 3: "MD5 校验失败"

**原因**：下载的文件损坏或不完整。

**解决方案**：

1. **删除损坏的文件**：
```bash
rm -rf artbench10_data
rm -f artbench-10-python.tar.gz
```

2. **重新下载**：
```bash
python download_artbench10.py
```

3. **如果仍然失败，尝试手动下载**：
   - 访问：https://artbench.eecs.berkeley.edu/files/artbench-10-python.tar.gz
   - 使用浏览器或下载管理器下载
   - 放置到 `model_training/` 目录
   - 运行：`python download_artbench10.py --verify`

---

### 错误 4: 内存不足

**错误信息**：
```
MemoryError 或 Out of memory
```

**解决方案**：

1. **减小批次大小**：

编辑 `train_with_artbench10.py`：
```python
BATCH_SIZE = 16  # 从 32 改为 16 或 8
```

2. **关闭其他应用**释放内存

3. **增加虚拟内存**（Windows）：
   - 设置 → 系统 → 关于 → 高级系统设置
   - 性能 → 设置 → 高级 → 虚拟内存
   - 增加到 8-16 GB

---

### 错误 5: 训练速度慢

**解决方案**：

1. **使用 GPU**（如果有）：

编辑 `train_with_artbench10.py`：
```python
context.set_context(
    mode=context.GRAPH_MODE,
    device_target="GPU",  # 改为 GPU
    save_graphs=False
)
```

2. **增加 worker 数量**：

```python
dataset = ds.Cifar10Dataset(
    dataset_dir=cifar_dir,
    usage="train" if is_training else "test",
    num_parallel_workers=8,  # 从 4 增加到 8
    shuffle=is_training
)
```

3. **启用 dataset_sink_mode**（如果支持）：

```python
model.train(
    epoch=EPOCHS,
    train_dataset=train_dataset,
    callbacks=callbacks,
    dataset_sink_mode=True  # 改为 True
)
```

data_path = "./artbench10_data/artbench-10-batches-py"

    dataset = ds.Cifar10Dataset(
        dataset_dir=data_path,
        usage="train"
    )

    print(f"  样本数量: {dataset.get_dataset_size()}")

        break
    
    print("✅ 测试通过！")
    
except Exception as e:
